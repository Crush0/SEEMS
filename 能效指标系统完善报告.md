# SEEMS 能效指标系统完善报告

## 📋 项目概述

根据智能船舶规范《2025》对纯电池动力推进船舶的要求，本次更新为SEEMS（船舶能效管理系统）完善了所有5个强制性能效及能耗指标的计算和展示功能。

## ✅ 完成的指标

### 1. 日耗电量 (`dailyEnergyConsumption`)
- **计算公式**: 基于电池SOC变化和电池容量计算日总电能消耗
- **数据来源**: BatteryLog 表中的SOC数据
- **更新频率**: 每小时自动更新
- **展示位置**: 数据分析页面 - 第一个卡片
- **单位**: kWh

### 2. 单位小时耗电量 (`preHourEnergyConsumption`)
- **计算公式**: 统计前一小时的电能消耗总量
- **数据来源**: BatteryLog 表，按小时聚合
- **更新频率**: 每小时自动计算
- **展示位置**: 数据分析页面 - 第二个卡片
- **单位**: kWh

### 3. 单位运输功电能消耗量 (`preUnitWorkEnergyConsumption`) 🆕
- **计算公式**: `电能消耗(kWh) / (拖力(t) × 航行距离(km))`
- **适用场景**: 针对纯电动拖轮的能效特性设计
- **数据来源**: 电能消耗数据 + 船舶拖力参数 + GPS航程数据
- **更新频率**: 每小时自动计算
- **展示位置**: 数据分析页面 - 第三个卡片
- **单位**: kWh/(t·km)

### 4. 单位航行距离电能消耗量 (`preDistanceEnergyConsumption`) 🔧
- **计算公式**: `电能消耗(kWh) / 航行距离(km)`
- **改进内容**: 新增专门计算方法，提高精度和可靠性
- **数据来源**: 电能消耗数据 + GPS航程数据
- **更新频率**: 每小时自动计算
- **展示位置**: 数据分析页面 - 第四个卡片
- **单位**: kWh/km

### 5. 航次/航段耗电量 (`voyagePowerConsumption`) 🆕
- **计算公式**: 基于航次期间电池SOC变化计算总电能消耗
- **数据来源**: VoyageLog + BatteryLog 表
- **统计维度**: 按航次聚合，支持历史查询
- **展示位置**: 数据分析页面 - 新增"航次能效统计"卡片
- **单位**: kWh

## 🚀 系统功能特性

### 实时性
- **自动更新**: 所有指标每小时自动计算更新
- **定时任务**: `@Scheduled(cron = "0 0 0/1 * * ?")` - 每小时执行
- **即时刷新**: 支持手动触发立即分析

### 数据准确性
- **高精度计算**: 使用 `BigDecimal` 确保财务级精度
- **多源数据融合**: 电池日志、GPS数据、船舶参数综合计算
- **异常处理**: 完善的参数验证和异常处理机制

### 用户体验
- **可视化展示**: 图表化展示历史趋势
- **多维度统计**: 支持日、小时、航次多维度分析
- **国际化支持**: 完整的中文界面和术语

## 📝 部署说明

### 数据库要求
- 确保数据库中包含船舶拖力参数 (`towing_force`)
- 电池日志表 (`battery_log`) 需要完整记录SOC数据
- GPS日志表 (`gps_log`) 需要记录航行轨迹

### 配置要求
- Spring Boot 配置文件中设置正确的数据库连接
- Redis 服务正常运行（用于缓存实时数据）
- 定时任务需要启用 (`@EnableScheduling`)

### 前端部署
```bash
cd seems-frontend
npm install
npm run build
```

### 后端部署
```bash
cd seems-backend
mvn clean package
java -jar target/SEEMS-0.0.1-SNAPSHOT.jar
```
